🎯 Skills：你的专业助手

Skills 是包含指令、脚本和资源的文件夹。Claude 会根据任务需要自动加载相关的 Skills。

📚

工作原理

Claude 扫描可用的 Skills，找到相关匹配项。元数据优先加载（约100个令牌），完整指令在需要时加载（<5k令牌）。

💡

适用场景

  • ✓ 公司品牌指南
  • ✓ 合规程序
  • ✓ Excel 公式技巧
  • ✓ 个人编码风格
🎁

核心优势

  • ✓ 高效可复用
  • ✓ 自动加载
  • ✓ 确保一致性
  • ✓ 不需反复说明

📌 实际例子

创建一个"品牌指南 Skill"，包含公司的色彩搭配、字体规则和版面规范。当 Claude 创建演示或文档时，它会自动应用这些标准——无需你每次都重新解释。

💬 Prompts：即时对话的力量

Prompts 是你在对话中直接提供给 Claude 的自然语言指令。它们是临时性、会话性的。

✓ 什么时候用 Prompts

  • • 一次性请求： 总结这篇文章
  • • 对话调整： 让语调更专业
  • • 即时背景： 分析这个数据并找出趋势
  • • 临时指令： 将其格式化为项目列表

💡 何时转换为 Skill

如果你发现自己在多个对话中反复输入同样的 prompt，那就是转换为 Skill 的信号。

例如：将"用 OWASP 标准审查代码漏洞"或"生成包括执行摘要的分析报告"转换为 Skills。

📁 Projects：持久工作区

Projects 是自包含的工作区，拥有自己的聊天历史和知识库。所有对话都可以访问项目内的共享知识。

🔑 核心特性

  • 📚
    持久背景知识

    每次对话都可以使用项目中的所有文档和背景

  • 🏢
    工作区分离

    为不同的工作分开维护上下文

  • 👥
    团队协作

    在团队和企业计划中分享知识和历史

  • ⚙️
    自定义指令

    项目级别的语调、视角和方法设置

📌 什么时候用 Projects

✓ 需要持久背景信息

背景知识应该指导每一个对话

✓ 多个相关对话

为不同的工作主题分开组织

✓ 团队需要共享上下文

分享知识库和聊天历史

例子： 为"Q4 产品发布"项目上传市场研究、竞争分析和产品规范。该项目中的每个对话都可以访问这些资源。

📊 Projects vs Skills

Projects 为特定工作体提供 Claude 持久背景——比如公司代码库或研究项目。而 Skills 则教 Claude 如何做某事——比如你的团队写作标准或代码审查流程。如果发现自己在多个 Projects 中复制同样的指令，那就该创建 Skill 了。

🤖 Subagents：专职助手

Subagents 是拥有自己上下文窗口、自定义系统提示和特定工具权限的专业 AI 助手。

🎯 适用场景

  • ✓ 复杂多步工作流
  • ✓ 需要专业化行为的任务
  • ✓ 并行处理多个不同类型的任务
  • ✓ 需要隔离上下文和权限
  • ✓ 基于 Claude Code 或 Agent SDK 构建

💡 核心优势

  • ✓ 独立的上下文窗口
  • ✓ 自定义系统提示
  • ✓ 特定的工具权限
  • ✓ 多任务并行处理
  • ✓ 专业化的行为

🔗 MCP：模型上下文协议

MCP（Model Context Protocol）是开放标准，允许 Claude 与外部系统、API 和数据源无缝集成。

🔌

连接性

将 Claude 连接到数据库、API、文件系统和其他工具——打破孤立。

🔄

互操作性

通过统一协议实现系统之间的无缝通信和数据交换。

⚡

实时集成

Claude 可以实时访问最新的工具、数据和资源。

🌐 MCP 在实践中的应用

📊 数据访问

直接连接公司数据库进行实时查询和分析

🛠️ 工具集成

集成 GitHub、Salesforce、Jira 等工具

📁 文件系统

访问和管理文件、文档和资源

🔐 安全连接

通过标准协议安全地连接企业系统

🚀 实战指南：如何选择合适的工具

📌

情景 1：快速写总结

你需要分析一份报告并生成摘要

→ 使用 Prompts

直接输入任务，获得即时结果

📌

情景 2：一致的品牌输出

需要 Claude 始终使用公司品牌指南创建内容

→ 创建 Skill

一次配置，永久使用。无需每次重复

📌

情景 3：长期项目管理

需要在整个产品发布项目中维护背景知识

→ 使用 Project

上传所有相关文档，所有对话都可访问

📌

情景 4：自动化复杂工作流

需要 Claude 自动处理多个专业领域的复杂任务

→ 使用 Subagents

创建专职 AI 助手，各司其职，协同工作

📌

情景 5：集成外部系统

需要 Claude 访问数据库、API 或其他工具的实时数据

→ 使用 MCP

通过标准协议连接外部系统

💎 最佳实践：组合使用所有工具

🎯 分层策略

  1. 1

    用 Skills 定义"怎么做"

    建立可复用的工作流和指导原则

  2. 2

    用 Projects 组织"背景信息"

    为不同的工作体维护持久知识库

  3. 3

    用 Subagents 处理"复杂任务"

    为多个专业领域部署专职助手

  4. 4

    用 MCP 连接"外部系统"

    实时访问数据和工具资源

  5. 5

    用 Prompts 进行"即时调整"

    在具体对话中进行微调和定制

✅ 核心原则

  • 🔄

    重复才创建 Skill

    发现自己多次输入相同指令时，转换为 Skill

  • 📚

    Projects 是背景，Skills 是行为

    区分"我需要知道什么"和"我该怎么做"

  • 🎯

    Prompts 永远重要

    即使有 Skills 和 Projects，Prompts 仍是关键交互方式

  • 🚀

    从简单开始

    先用 Prompts，再根据需要升级到 Skills 和 Projects

🎬 真实场景：完整工作流

场景： 管理一个年度产品发布项目，需要整个团队协作生成营销文案

  1. 1. 创建 Project： "年度产品发布"，上传市场研究、竞争分析、产品规范
  2. 2. 创建 Skill： "品牌写作指南"，定义语调、风格、关键信息
  3. 3. 部署 Subagent： 创建"文案撰写助手"自动生成营销内容
  4. 4. 集成 MCP： 连接 Salesforce 数据库，获取实时销售数据
  5. 5. 使用 Prompts： 对生成的内容进行具体调整："让这段文案更吸引年轻用户"

🎓 核心收获

💬

Prompts 是基础

一次性对话指令，永远是与 Claude 交互的主要方式

🎯

Skills 提升效率

重复任务才创建 Skill，自动应用可复用专长

📁

Projects 组织知识

维护持久背景，为不同工作体分离上下文

🤖

Subagents 自动化

复杂工作流交给专职 AI 助手处理

🔗

MCP 无限扩展

连接外部系统，实时访问数据和工具

🚀

从简到繁

从 Prompts 开始，逐步扩展到其他工具

Claude 的工具栈设计得非常清晰：Prompts 是基础，所有其他工具都是为了让你更高效地使用 Claude。选择合适的工具组合，能够显著提升工作效率和质量。
