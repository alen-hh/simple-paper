为什么需要 Agent Skills？

AI 助手虽然强大，但在真实工作中往往缺少必要的专业知识

❌

没有 Skills 的情况

  • • 每次都得重新解释工作流程
  • • AI 不懂公司特定的规则和格式
  • • 需要为不同任务重新编写 AI 指令
  • • 容易出错，无法复用经验
✅

使用 Skills 后

  • • AI 自动加载相关的专业知识
  • • 理解组织的规则和最佳实践
  • • 一次创建，处处使用
  • • 专业化的 AI 助手，效率更高

Agent Skills 是什么？

简单来说，就是给 AI 助手的"工作手册"

Agent Skills 是一个文件夹，包含：

📄

指导说明

清晰的步骤和规则，告诉 AI 该怎么做

⚙️

脚本代码

可执行的工具和脚本，完成具体操作

📦

资源文件

必要的参考材料和数据资源

Skills 的核心特性

🎯

可组合

多个 Skills 可以组合使用，AI 自动识别需要哪些

📦

可移植

在各个 Claude 产品间通用，一次创建处处使用

⚡

高效

只加载需要的部分，节省 AI 上下文空间

💪

强大

不仅有指导，还能执行代码，精准可靠

Skills 的构成

层层递进的信息架构，按需加载

三个层级

1️⃣

名字和描述

AI 一开始看到的，用来判断是否需要这个 Skill

2️⃣

完整说明文件

Skill 被激活后，AI 会读取 SKILL.md 了解具体内容

3️⃣

附加资源

特定场景的文件（如表单、代码等），按需加载

实际例子：PDF Skills

pdf-skill/
📄 SKILL.md
↳ 基本说明（文件处理方法）
📄 reference.md
↳ 详细参考（高级功能）
📄 forms.md
↳ 表单处理指南（填写 PDF 表单）
⚙️ extract_fields.py
↳ 可执行脚本（提取表单字段）

💡 这样设计的好处：Skills 不会过于庞大，AI 只在需要时才加载额外信息

工作流程

AI 如何发现和使用 Skills

为什么这样设计？

  • ✓ 节省 AI 的"思考空间"（上下文窗口）
  • ✓ Skills 可以无限大，不受限制
  • ✓ 速度更快，响应更迅速
  • ✓ 用户体验更好

何时使用代码执行？

某些操作用代码比用文字描述更好：

  • 🔧 排序、数据处理等确定性操作
  • 🔧 性能要求高的任务
  • 🔧 需要绝对准确的操作

如何创建好的 Skill

创建有效 Skills 的最佳实践

🎯

1. 从问题开始

先用 AI 完成实际任务，观察哪里出现问题，然后针对性地创建 Skill 解决这些问题

📦

2. 结构要清晰

当 SKILL.md 太长时，把不同部分拆成单独的文件。这样 AI 只在需要时加载特定内容

🤖

3. 从 AI 的角度思考

Skill 的名字和描述很重要，这直接影响 AI 何时使用它。名字要清晰，描述要准确

🔄

4. 持续迭代

和 AI 一起工作时，观察它的行为。如果 Skill 哪里用得不好，要求 AI 分析原因并改进

🔐

5. 安全第一

只使用来自可信来源的 Skills。安装陌生的 Skill 前要仔细检查其内容，特别是代码部分

⚖️

6. 代码 vs 文字

确定性任务用代码执行更可靠，建议和参考信息放在文字中让 AI 理解

安全性考虑

⚠️ 只从可信来源安装 Skills

Skills 给了 AI 执行代码的能力。恶意 Skill 可能：

  • • 访问你的计算机文件
  • • 连接到不安全的网络源
  • • 泄露敏感数据

💡 检查 Skill 的代码、依赖关系和网络连接，确保安全后再使用
